## 项目简介

**Real World Danmaku AI** 是一个将现实世界实时影像与 AI 生成的短视频内容深度融合的实验性 Web 应用。打开浏览器摄像头，AI 视觉模型实时分析每一帧画面，生成：

- 🎬 **弹幕** — 基于 12 种幽默结构的抖音风格弹幕，带有损友感的尖锐吐槽
- 🎯 **挑战** — AI 根据画面内容即时生成单动作互动挑战，并自动验收评分
- 🎥 **导演方案** — 整合弹幕与挑战上下文，输出短视频拍摄方案并一键导出竖屏视频

核心理念：**让现实世界像短视频一样被弹幕包围**——摄像头不再是单纯的记录工具，而是一个实时内容创作的舞台。

## 核心功能

| 功能模块 | 描述 |
|---------|------|
| 实时画面分析 | 摄像头每隔 N 毫秒（最低 300ms）截取当前画面，发送至视觉语言模型（VLM）分析 |
| 弹幕飞屏 | 生成 8–18 字抖音式弹幕，在画面层以飞屏动画呈现，支持热度染色 |
| 幽默结构引擎 | 内置 12 种原创幽默结构（物体拟人、身份错位、功能反转等），确保弹幕风格多样且不重复 |
| 内容安全过滤 | 前端 + 后端双重过滤，自动替换人身攻击、低俗词汇为空泛吐槽，保留尖锐感但不出格 |
| 互动挑战 | AI 根据画面中可见物体、空间、人物姿势，生成 15 秒内可完成的单动作挑战 |
| 自动验收 | 挑战模式下按固定频率抽帧，AI 逐帧打分（0–100），80 分以上自动通关并切换下一挑战 |
| 挑战去重 | 多维度检测挑战相似度（文本归一化 + 语义家族聚类），避免连续生成同类动作 |
| 最高分画面 | 记录挑战过程中的最高分瞬间截图，可用于封面或分享 |
| 导演方案 | AI 结合最近弹幕和挑战上下文，输出封面标题、开场白、镜头建议、字幕、结尾互动话术 |
| BGM 智能匹配 | 视觉模型判断画面氛围 → 生成搜索词 → 调用网易云音乐 API 搜索候选热歌 |
| 短视频导出 | Canvas 实时渲染 1080×1920 竖屏画面，内置 AI 生成的节拍音轨，导出 10 秒 WebM 视频 |
| 请求并发控制 | 前端请求限流器，最多 2 个并发 AI 请求，避免 API 过载与重复提交 |
| 场景上下文 | 用户可输入全局场景描述（如"这是比赛现场"），所有模式共享该上下文，提升分析准确性 |

**数据流：**
1. 摄像头流 → Canvas 截图（JPEG base64）
2. 截图 + 场景上下文 + 模式提示词 → POST 到后端
3. 后端构造完整的 VLM 对话（system prompt + user prompt + image）→ 调用 API
4. VLM 返回 JSON → 后端解析、规范化、兜底处理 → 返回前端
5. 前端渲染弹幕动画 / 挑战卡片 / 导演方案 / BGM 候选列表

## 三种工作台

### 1. 弹幕工作台 (Danmaku)

> 实时生成抖音式弹幕，在你的视频画面上飞舞。

- **可调参数：**
  - 弹幕间隔：300ms – 5000ms
  - 弹幕密度：1–3 档（实际生成 `档位 × 3` 条弹幕）
- **弹幕列表：** 最近 20 条弹幕以聊天样式展示，带虚拟用户头像和昵称
- **幽默引擎：** 每批弹幕至少覆盖 4 种幽默结构，至少半数带有损友感的尖锐吐槽
- **内容安全：** 自动过滤攻击颜值、身材、种族、粗口等违规内容，替换为机智的空泛吐槽

**12 种内置幽默结构：**

| # | 结构名称 | 逻辑 | 示例 |
|---|---------|------|------|
| 1 | 物体拟人 | 给物体情绪、立场 | "电脑也在等下班" |
| 2 | 身份错位 | 把普通东西安排成离谱职位 | "咖啡是项目延期保险" |
| 3 | 功能反转 | 交换人和工具的控制关系 | "不是人在用电脑，是电脑在管理人" |
| 4 | 字面误解 | 故意按字面曲解动作或名称 | "扫地机器人正在给地板做绩效面谈" |
| 5 | 因果倒置 | 把结果一本正经说成原因 | "不是桌面乱，是文件在争取曝光" |
| 6 | 一本正经夸大 | 用严肃口吻升级普通小事 | "这一步操作足以载入公司野史" |
| 7 | 职场类比 | 套用汇报、绩效、背锅逻辑 | "键盘负责输出，人负责背锅" |
| 8 | 游戏术语类比 | 用副本、技能、掉线等表达 | "这个操作把新手教程打成隐藏副本" |
| 9 | 系统状态类比 | 把现场描述成报错/系统状态 | "当前理智进程已停止响应" |
| 10 | 伪哲理 | 从小事推出荒谬人生结论 | "桌面越满，说明空闲越抽象" |
| 11 | 直播观众反应 | 模拟围观群众即时补刀 | "观众席已经替这个操作尴尬完了" |
| 12 | 前提拆解 | 接受前提再指出离谱漏洞 | "思路没问题，问题是它只有思路" |

### 2. 挑战工作台 (Challenge)

> AI 根据画面生成互动挑战，自动验收评分，达标自动换题。

- **挑战生成：** 结合画面中可见物体、空间、人物姿势，从 8 种热门挑战机制中选取创意
- **单一动作约束：** 每个挑战只要求一个可在一张抽帧中验收的动作（不允许多步骤、组合动作）
- **智能去重：** 文本归一化 + 12 类语义家族聚类，避免连续重复同类挑战（如"举手到头"后不会出"摸头"）
- **自动验收：** 每 800ms 抽帧 → VLM 打分（0–100），80 分以上触发通关特效 → 自动进入下一挑战
- **最高分记录：** 持久化存储每个挑战的最高分瞬间画面
- **跳过机制：** 用户可跳过不喜欢的挑战，AI 会将跳过记录纳入上下文避免再次生成
- **备用题库：** 内置 7 个基础挑战模板，当 AI 返回重复挑战时自动切换到不重复的备用题

### 3. 导演工作台 (Director)

> 整合弹幕和挑战上下文，生成短视频方案，匹配 BGM，一键导出。

- **方案生成：** 封面标题、开场白、镜头建议、字幕文案、结尾互动
- **BGM 匹配流程：**
  1. 视觉模型分析画面氛围（搞笑 / 日常 / 治愈 / 热血 / 卡点 / 赛博 / 打工人等）
  2. 生成 4–6 个中文搜索关键词
  3. 调用网易云音乐搜索 API 获取候选歌曲（歌名、歌手、专辑、时长）
  4. 前端展示候选列表，用户可选择 BGM
- **短视频导出：**
  - 1080×1920 竖屏（9:16）Canvas 实时渲染
  - 弹幕以滚动字幕形式包含在视频中
  - 封面标题、导演方案、挑战信息以短视频排版布局呈现
  - 内置 AI 原创节拍音轨（126 BPM，多音色编曲）— 无需外部音频文件，避免版权问题
  - 选中 BGM 的标签信息写入视频
  - 导出 10 秒 WebM 格式（VP9 编码），可直接分享

## 技术亮点

### 1. 多层次内容安全架构

```
VLM Prompt 层 → 明确禁止攻击颜值、身材、种族等
     ↓
JSON 结构化输出 → 限制字段长度与格式
     ↓
sanitizeDanmakuList → 正则匹配 + 智能替换
     ↓
validateHumorStructures → 启动时校验幽默结构库完整性
```

### 2. 请求并发控制

前端 `request-limiter.js` 实现基于 Promise 的信号量限流器：
- 弹幕模式最多 2 个并发请求
- 超限请求直接丢弃（返回 `null`），不排队积压
- 失败自动释放槽位
- 前端 `busy` 标志位防止状态竞争

### 3. 挑战去重系统

多维相似度检测，防止 AI 连续生成"换皮"挑战：
- **文本归一化：** 去空格、去标点、全小写
- **语义家族：** 12 个预定义同义动作聚类（"头顶/摸头/举手到头"归为一类）
- **历史记忆：** 记住最近 6 个已生成的挑战 + 1 个刚跳过的挑战

### 4. 弹幕动画系统

- 多轨道布局（根据画面高度动态计算轨道数 `≥5`）
- CSS `@keyframes` 飞屏动画，0.75× 慢速比让弹幕更易读
- 首条弹幕高亮（金黄色 `hot` 状态）
- 批量投放（120ms 间隔错峰出现）

### 5. 健壮的 JSON 解析

- 自动去除 Markdown 代码块（` ```json ``` `）
- 正则兜底提取 JSON 对象
- 非 JSON 输出自动转换为兜底弹幕，不中断服务

### 6. 短视频编曲引擎

- 客户端 Web Audio API / 手动 WAV 编码双路径支持
- 多音色编曲：旋律（三角波）、贝斯（正弦波）、底鼓、踩镲
- 126 BPM 自动卡点，10 秒时长
- 渐变淡出结尾

## 安装与运行

### 前置条件

- Node.js ≥ 18
- 一个支持视觉输入（图片 → 文本）的 VLM API Key（兼容 OpenAI `/chat/completions` 格式）
- 现代浏览器（Chrome / Edge，支持 `getUserMedia` 和 `MediaRecorder`）

### 安装

```bash
git clone <repo-url>
cd douyinProject
npm install
cp .env.example .env
```

### 配置 `.env`

```env
OPENAI_API_KEY=你的API密钥
OPENAI_BASE_URL=https://your-proxy.com/api/v1
OPENAI_MODEL=gpt-4o-mini
PORT=3000
```

> 如果中转平台提供完整的 `/chat/completions` 地址，也可以直接设置 `OPENAI_CHAT_COMPLETIONS_URL`，它会优先于 `OPENAI_BASE_URL`。

### 运行

```bash
npm start        # 生产模式
npm run dev      # 开发模式（文件变更自动重启）
```

打开浏览器访问 `http://localhost:3000`，点击"开启摄像头"即可开始。

### 注意事项

- `localhost` 下浏览器自动允许摄像头权限，无需 HTTPS
- 必须选择支持图片输入的视觉模型（如 `gpt-4o-mini`、`gpt-5.4`）
- API 响应时间决定弹幕刷新频率，建议分析间隔 ≥ 模型平均响应时间
- BGM 功能仅搜索和推荐，不下载平台音频；导出视频使用原创节拍音轨

## API 说明

### `GET /api/health`

检测后端状态和 API Key 配置。

**响应：**
```json
{
  "ok": true,
  "model": "gpt-4o-mini",
  "baseUrl": "https://your-proxy.com/api/v1",
  "hasApiKey": true
}
```

### `POST /api/analyze-frame`

分析摄像头画面，根据模式生成对应内容。

**请求体：**
```json
{
  "image": "data:image/jpeg;base64,...",
  "mode": "danmaku | challenge | director",
  "density": 2,
  "sceneContext": "用户描述的场景文本",
  "validationChallenge": "验收挑战时的挑战描述文本"
}
```

**响应（弹幕模式示例）：**
```json
{
  "mode": "danmaku",
  "title": "桌面上的赛博战场",
  "hook": "这工位比想象中有故事",
  "danmaku": ["键盘负责输出，人负责背锅", "这杯咖啡是项目延期保险"],
  "scene_tags": ["办公桌", "咖啡", "电脑"],
  "heat": 72,
  "challenge": {},
  "director": {},
  "confidence": 0.85
}
```

### `POST /api/search-bgm`

视觉分析画面氛围并联网搜索匹配的 BGM 候选。

**请求体：**
```json
{
  "image": "data:image/jpeg;base64,...",
  "sceneContext": "全局场景描述",
  "title": "当前标题",
  "hook": "当前Hook",
  "directorText": "导演方案文本",
  "challengeText": "当前挑战文本"
}
```

**响应：**
```json
{
  "scene": "桌面上的咖啡和电脑",
  "mood": "打工人日常",
  "reason": "适合轻松吐槽风格的日常BGM",
  "search_terms": ["打工人BGM", "日常搞笑背景音乐", "轻松卡点"],
  "style_tags": ["日常", "轻松", "卡点"],
  "candidates": [
    {
      "id": 123456,
      "title": "歌曲名",
      "artist": "歌手",
      "album": "专辑",
      "duration": "3:45",
      "source": "网易云音乐搜索",
      "sourceUrl": "https://music.163.com/#/song?id=123456"
    }
  ]
}
```

## 设计理念

### 1. "零依赖前端"

前端不引入任何第三方框架或库。所有 UI 交互、状态管理、动画渲染、视频导出均为原生实现。这确保了：
- 极快的首屏加载速度
- 零供应链安全风险
- 代码完全可控、可审计

### 2. "AI 作为内容引擎，而非工具"

传统 AI 应用将模型视为问答工具；本项目中 VLM 是内容生产引擎——它根据画面"即兴创作"弹幕、挑战和导演方案。提示词工程（12 种幽默结构 + 8 种挑战机制）是这个引擎的核心。

### 3. "现实即舞台"

摄像头不再是视频通话的工具，而是把任何场景变成短视频内容的入口。桌面、会议室、厨房、街头——任何地方都可以"被弹幕"。

### 4. "安全地有趣"

幽默不等于攻击。多层内容安全机制确保 AI 输出保持机智和损友感，但不触碰人身攻击、歧视、低俗等红线。

## 未来方向

- [ ] **多模态增强**：接入语音输入，允许用户用语音描述场景或回复弹幕
- [ ] **多人协作**：多设备同时连接，弹幕合并显示（适合活动现场）
- [ ] **直播推流**：对接直播平台 RTMP 推流，弹幕与真实直播弹幕融合
- [ ] **历史回放**：保存分析历史，支持回放和编辑
- [ ] **更多 BGM 源**：接入 QQ 音乐、抖音热歌排行榜等更多音乐平台
- [ ] **挑战排行榜**：多人模式下的挑战得分排名
- [ ] **自定义幽默结构**：允许用户创建和分享自己的幽默结构模板
